# 侧键链接复制器（Side Button Link Copier）

[English](./README.en.md) | 中文

通过鼠标侧键（后退/前进）快速批量复制页面上结构相似的链接，适合爬取列表页、搜索结果、商品卡片等场景。

## 功能

- **后退键（鼠标侧键 3）**：复制**所有相似结构**的链接  
  基于当前链接的通用 CSS 路径（忽略位置差异），选中页面上所有结构相同的 `<a>` 标签，批量复制它们的 `href`。

- **前进键（鼠标侧键 4）**：复制**相同位置**的链接  
  在通用 CSS 路径中，为最近一个存在多个同标签兄弟的分支添加 `:nth-child` 索引，只选中与当前链接位置相同的元素。

## 安装

1. 安装浏览器扩展 **Tampermonkey**（支持 Chrome / Firefox / Edge 等）。
2. 新建用户脚本，将本脚本代码粘贴进去并保存。
3. 确保脚本状态为“已启用”。

## 使用说明

1. 鼠标移动到任意一个你想要作为“模板”的链接上。
2. 按下鼠标侧键：
   - **后退键（`←`）**：批量复制所有与该链接结构相似的链接地址。
   - **前进键（`→`）**：批量复制与该链接**位置相同**的链接地址（即页面上对应位置的一组链接，例如每个列表的第一项）。
3. 所有链接会以换行分隔复制到系统剪贴板。
4. 页面中所有被复制到的链接会短暂高亮为红色边框（0.5 秒），同时弹出通知显示复制数量。

## 示例场景

| 场景 | 推荐操作 |
|------|----------|
| 搜索结果页中，需要复制所有结果的链接 | 鼠标悬停任意一个结果链接 → **后退键** |
| 商品列表页中，只需要复制每个商品的“购买链接”（但每个商品有多个链接） | 鼠标悬停第一个商品的“购买链接” → **前进键** |
| 表格或卡片列表中，复制每行第二列的链接 | 悬停某个第二列的链接 → **前进键**（因为位置相同） |

## 工作原理

脚本生成 CSS 选择器时采用了两种策略：

1. **不区分位置（后退键）**  
   路径中不添加 `:nth-child`，得到的路径会匹配页面上所有相同标签+相同 class 链路的元素。  
   例如：`div.item a.title` → 匹配所有 `.item` 内的 `.title` 链接。

2. **区分位置（前进键）**  
   在最近的一个存在多个同标签兄弟的层级上添加 `:nth-child(index)`，从而更精确匹配位置。  
   例如：`div.list > a:nth-child(2) .btn` → 只匹配每个列表的第二项中的按钮链接。

## 注意事项

- **侧键默认行为被覆盖**：后退/前进键不再执行浏览器页面导航，而是触发复制功能。如果你习惯用侧键翻页，请谨慎使用或临时禁用脚本。
- 只对 `<a>` 标签有效（鼠标必须悬停或点击到链接或其内部元素上）。
- 如果相似结构数量为 0 或所有链接 `href` 为空，脚本会给出失败提示。
- 复制结果仅包含 `href` 属性（绝对 URL），不含文本或其他内容。
- 需要 Tampermonkey 授予 `GM_setClipboard` 和 `GM_notification` 权限（脚本开头已声明）。

## 兼容性

- 适用所有 `http://` 和 `https://` 页面（`@match *://*/*`）。
- 不依赖第三方库，原生 JavaScript 实现。
- 理论上兼容绝大多数现代浏览器（Chrome 70+ / Firefox 65+ / Edge 80+）。

## 常见问题

**Q：按下侧键没有反应？**  
A：检查鼠标侧键是否工作正常；尝试按下时控制台有无报错；确保脚本已启用且页面完全加载。

**Q：复制到的链接比预期多或少？**  
A：这是因为生成的 CSS 选择器范围不精确。可以尝试改用前进键（位置区分）来缩小范围；或者调整页面结构，给链接所在的容器增加唯一的 class。

**Q：如何临时恢复侧键的导航功能？**  
A：在 Tampermonkey 菜单中禁用该脚本，或修改脚本代码删除 `e.preventDefault()` 部分。
